Identifying Duplicate Cases and Unusual Cases

Identifying duplicate cases and unusual cases are two options in SPSS which are of immense use. Both duplicate cases and Unusual Cases can have serious impact on the analysis and the subsequent output. It is very important to identify duplicate and unusual cases and take appropriate actions to remove these cases or minimize its impact, on the analysis.

Identifying Duplicate Cases

Duplicate cases are when cases are identical to already existing cases. This can happen when the data was purposively or inadvertently copied. When the data file is very large, it will be difficult to identify such duplicate cases by observation. SPSS will identify the cases which are duplicates and will display in the output viewer. In the SPSS output file, all original data will be displayed as Primary Case and duplicate cases as Duplicate Case. To illustrate this, first step will to run without duplicate cases on the data file.
Data > Identify Duplicate Cases In the box Define matching cases by: move all the relevant variables that the user would like to use, to identify duplicates.

The output viewer will show the following results. It is clear from the output that all cases have been listed as Primary. This implies that all cases are original and there are no duplicate cases. Identifying unusual cases

When the response of any case, is very different from the responses obtained from similar cases it is termed as an unusual case. It is also referred to as outlier or anomaly case. In statistics, outliers are defined as a data values that is significantly different from other data values. It can also happen due to error committed during data entry, variability in measurement of data, experimental error etc. Outliers causes severe problems in analysis of data and so if identified, are excluded from statistical analysis.

For example in the DataSetl.sav the data entry operator, has wrongly entered the age of George (Case 2) as 91 instead of 19. This is an unusual case and should be identified and rectified, if not, the analysis will be based on age as 91.

Sort Order

Sorting Cases Sorting cases is arranging data in the ascending or descending order. Data can be sorted in ascending order (from A to Z) or in descending order (from Z to A). In case of numeric data, the user can sort in ascending order (lowest to highest) or in descending order (from highest to lowest). After sorting, the user can also save file with the sorted data. Data> Sort data.

Sort by dialogue box will appear.

If the user wants to sort data based on age, the age of the respondents will have to be moved into the Sort by box. Click on Sort order Ascending radio button Click OK

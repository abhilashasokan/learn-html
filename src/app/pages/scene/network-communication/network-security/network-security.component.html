Network Security

Network security refers to the steps and policies adopter by the network administrator to protect the network and the network-accessible resources from unauthorised access. Only legal and authorised users are given access to the network. Network security starts from authenticating the user. commonly with a username and a password. Once authenticated, a firewall enforces access policies which determine the access rights of network users.

  • 'Authorising' the users to gain access to network resources.
  • 'Authenticating' authorised users to use approved devices. Authentication is the process of verifying that information is coming from a trusted source.
  • Implement a proper 'Security Policy' in the network.
  • Establish 'digital signature' which is a way to ensure that electronic document (email, text file or spreadsheet etc.) is authentic.
  • 'Install antivirus software' in all servers and workstation
  • Install a strong Firewall?, which is software or hardware or both, protecting against unauthorised access.
  • Set strong account lookout policy.
  • All the data that goes into the server should be backed up and restored.
  • Perform network security audit for fixing security issues.
  • 'Monitor the users' activities on the internet and block all the unwanted websites and web applications which have security risks.

Firewalls

A firewall can be defined as a hardware and/ or software placed between an organisations internal network and external network. Firewall determines which inside services may be accessed from outside, and which may be accessed from inside. Firewall is normally placed between the network gateway and the server. There are two types of firewalls namely:

  • Inside Firewall: Internal firewall protects the network and machines of the intranet of the company.
  • Outside Firewall: protects the access between the company's networks with that of all the other networks (extranets). All traffic from the outside must pass through it.

Firewalls

Without a firewall a company's system are exposed to insecure services and to probes and attacks from hosts elsewhere on the network. The major benefits are enumerated below

  1. Deny unauthorised sources from connecting into the network
  2. Deny connections from inside the network to unauthorised sources or services outside.
  3. Maintain a log of usage statistics.
  4. Hide internal private network in case of emergency.
